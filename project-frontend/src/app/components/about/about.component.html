Login and Profile

  • When users register their passwords are confirmed as matching and complex. This new user is then checked in the backend to ensure no duplicate usernames or emails are registered
  • Once registered users are prompted to login with their new credentials
  • On login the username and a token are saved in local storage. This token is valid for 1 hour, after which time the user will need to log back in
  • The profile is displayed as a list of current values from the database
  • The interests are retrieved as an Object of boolean key pairs that are converted into an array and displayed as a string
  • When user clicks edit a form is revealed that allows user to alter their profile and select new interests
  • On save this is updated in the back end and the new data is displayed

Voting

  • The voting portion of our website can be found in the user dropdown once logged in under Fundraising.
  • There are 2 separate databases linked to this section
    • There is a current voting database which allows users and admins to see what the current voting ideas status is
    • There is also a New Idea database that allows users to suggest ideas for forthcoming events
    • Both these databases have validation in the back end to ensure duplicate names and short names are not suggested
  • In the user model the is an attribute 'isAdmin', this is only set to true by the users who run the site and can be manually adjusted in mLab only.
  • As an admin the user has different options to the regular users
    • Under Current Votes they can see the current tally of user votes and select a winner
    • Below this section they have the ability to delete current voting options entirely
    • They can also reset votes so the idea can be used again
    • By selecting New Vote they can populate the Current Voting options with a brand new idea
    • Alternatively they can browse suggested ideas below and either delete or edit them - editing pulls the details up into the form for submission
  • Users have much less control over the databases
    • They can vote for one of the current ideas suggested by the admins
    • Once voted they get to see a graph of the current voting status
    • They can also suggest a new idea for forthcoming events
